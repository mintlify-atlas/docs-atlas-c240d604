# NOUS Academy

A complete desktop management system for private academies. NOUS Academy lets you manage users, schedule and run classes, assign memberships and contracts, and track academy finances — all from a role-based Java Swing interface.

## Three roles, one platform

<Tabs>
  <Tab title="Administrator">
    Administrators have full access to the system:
    - Create, update, and delete students, teachers, and other admins
    - Assign and update student memberships
    - Hire teachers and manage their contracts
    - Create and manage groups and reservations
    - Inject or withdraw academy capital
  </Tab>
  <Tab title="Student">
    Students interact with classes and groups:
    - Join and leave groups
    - Book and cancel private class reservations (requires an active membership)
    - Join in-progress group sessions
    - Rate completed classes
  </Tab>
  <Tab title="Teacher">
    Teachers manage their class sessions:
    - View assigned reservations
    - Start a class (triggers attendance check)
    - Finish a class (triggers ratings window for students)
  </Tab>
</Tabs>

## Key features

<AccordionGroup>
  <Accordion title="Multi-role user management">
    Three distinct roles — ADMINISTRADOR, ESTUDIANTE, DOCENTE — each with a dedicated GUI frame and permission set. All validation (email format, document length, name length, phone sign) is enforced at the model layer.
  </Accordion>
  <Accordion title="Flexible membership system">
    Four membership variants: BASICA ($1.2M/month, no private classes), INTENSIVA ($2.2M/month, 4 private classes), PREMIUM ($3.5M/month, 8 private classes), and PERSONALIZADA (fully configurable price and private-class quota). Membership status transitions between EN_PROGRESO, SUSPENDIDA, and CANCELADA.
  </Accordion>
  <Accordion title="Full class lifecycle">
    Reservations move through states: scheduled → teacher assigned → IN_PROGRESS (started) → FINALIZADA (finished) → rateable. Business rules enforce minimum attendance before starting, adequate duration before finishing, and per-student rating limits.
  </Accordion>
  <Accordion title="File-based persistence">
    No external database required. Every entity is Java-serialized to disk on every write. Administrator files use `.usr`, students `.est`, teachers `.doce`, groups `.grp`, reservations `.res`, and the academy configuration uses a plain text `.txt` file.
  </Accordion>
  <Accordion title="Capital tracking">
    Academy capital is updated automatically when memberships are sold (positive impact) and when teacher contracts are signed (negative impact equal to salary × months). Manual injection and withdrawal are also available.
  </Accordion>
</AccordionGroup>

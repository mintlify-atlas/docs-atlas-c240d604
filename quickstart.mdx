## Prerequisites

- **Java 11 or later** — the application is compiled for the standard Java SE platform.
- The `datos/` directory must exist next to the compiled classes (or the project root when running from an IDE). The boot classes write and read files there.

## Running the application

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/DuqueJR/NOUS_ACADEMY.git
    cd NOUS_ACADEMY
    ```
  </Step>
  <Step title="Compile the source">
    The project uses plain `javac`. Compile everything under `src/`:
    ```bash
    javac -d out -sourcepath src $(find src -name "*.java")
    ```
  </Step>
  <Step title="Create the data directory">
    ```bash
    mkdir -p datos
    ```
  </Step>
  <Step title="Launch the application">
    Run the Swing entry point:
    ```bash
    java -cp out com.nousacademy.interfacegrafica.NousSwingApp
    ```
    The login window opens. The boot sequence loads (or creates) the academy data from `datos/NousAcademy.txt` and any serialized user/group/reservation files it finds.
  </Step>
</Steps>

## First-time setup: bootstrap the academy

When no data files exist, `BootEmpresa` initializes the academy and `BootDeUsarioAdmin` seeds the first administrator account.

The default academy data loaded from `datos/NousAcademy.txt` follows this format:

```text datos/NousAcademy.txt
5422994301
EMPRESA
1234
@nousacademy.co
0.0
```

Fields (one per line): NIT · name · phone · email · capital.

<Note>
  On first run, ensure `datos/NousAcademy.txt` exists with your academy's NIT and name. If the file is missing the boot class creates it with default values.
</Note>

## Log in as an administrator

After launch the **Login** screen appears. Enter the administrator credentials seeded by `BootDeUsarioAdmin`. Once authenticated you are taken to **AdminFrame**, the main management console.

## Complete a basic workflow

<Steps>
  <Step title="Create a student">
    In **AdminFrame**, open the **Students** tab and click **New Student**. Fill in:
    - Name and last name (minimum 2 characters each)
    - Phone number (minimum 5 digits, positive)
    - Email (must end with `@nousacademy.co`)
    - Document type and number (minimum 5 digits)
    - Username and password

    The record is saved to `datos/<prefix><document>.est`.
  </Step>
  <Step title="Assign a membership">
    Select the newly created student and click **Assign Membership**. Choose one of the predefined tiers:

    | Tier | Private classes / month | Monthly price |
    |---|---|---|
    | Basic | 0 | $1,200,000 |
    | Intensive | 4 | $2,200,000 |
    | Premium | 8 | $3,500,000 |

    Or configure a fully custom membership with your own price and private-class quota.
  </Step>
  <Step title="Create a teacher">
    Open the **Teachers** tab, click **New Teacher**, and fill in the same personal fields plus an **Especialidad** (area of expertise). Then assign a contract specifying start date, base salary, minimum hours, additional-hour rate, and duration in months.
  </Step>
  <Step title="Create a group class reservation">
    In the **Reservations** tab, click **New Group Reservation**. Set the expected start time, title, description, topic, duration (minutes), and minimum/maximum attendance.
  </Step>
  <Step title="Assign the teacher to the reservation">
    Select the reservation and the teacher, then click **Assign Teacher**. The teacher record and reservation are both updated on disk.
  </Step>
  <Step title="Student joins and the class runs">
    Log in as the student. From **StudentFrame**, join the reservation. When the teacher logs in as **TeacherFrame**, they can start and finish the class. After it finishes, the student can rate it (1–5).
  </Step>
</Steps>
